## 2. 修复内容

问题原因：

- `seedance1.5-sr`、`seedance2-sr`、`seedance2.0fast-sr` 的 720p 链路内部是 `480p -> 720p`。
- 上游视频超分算子按宽度等比放大，源视频 `864x496` 放大后会得到 `1280x736`。
- 即使提交 `target_height=720`，上游超分仍返回 `1280x736`，说明超分算子不能保证严格目标高度。

修复方案：

- SR 提交状态中记录 `target_width` 和 `target_height`。
- SR 720p 完成后，自动追加一次 LAS 视频分辨率归一化处理，强制输出 `1280x720`。
- 最终成功 URL 指向 `/video-sr/normalized/` 下的归一化视频。
- 用户计费仍按原模型计费，不因内部归一化步骤额外向用户收费。
- dedicated `*-sr` 模型继续隐藏内部超分/归一化细节，前端仍按普通 720p/1080p 展示。

## 6. 后续建议

- 新增视频模型上线前，必须至少覆盖：计费倍率、预扣/实扣/退款差额、用户请求分辨率、内部输入分辨率、最终输出宽高、失败退款。
- 对任何 `*-sr` 隐藏超分模型，必须额外验证“前端展示分辨率”和“内部基座分辨率”是否按产品规则分离。
- 如果上游未来支持严格目标宽高，可保留归一化作为兜底；只有连续多轮确认上游稳定输出严格尺寸后，再考虑关闭兜底。
