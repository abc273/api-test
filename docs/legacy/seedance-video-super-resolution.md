# Seedance 视频超分说明（用户版）

## 1. 这是什么

现在视频超分不再通过给普通模型额外传参数来触发。

系统新增了 2 个专用超分模型：

- `seedance2-sr`
- `sd2.0fast-sr`

调用这两个模型时，系统会在原始视频生成完成后，自动继续执行视频超分处理。

普通模型仍然保持原样：

- `seedance2`：普通版，不自动超分
- `sd2.0fast`：普通版，不自动超分

## 2. 分辨率提升规则

当前规则如下：

| 提交分辨率 | 最终返回分辨率 |
| --- | --- |
| `480p` | `720p` |
| `720p` | `1080p` |
| `1080p` | 仍为 `1080p`，不再继续超分 |

## 3. 你需要关注什么

1. 超分能力现在由模型决定，不再依赖 `metadata.enable_video_super_resolution`
2. 如果调用普通模型 `seedance2` / `sd2.0fast`，返回的是原始生成视频
3. 如果调用专用超分模型 `seedance2-sr` / `sd2.0fast-sr`，最终 `result_url` 和 `/content` 返回的是超分后成品视频
4. 专用超分模型属于付费增值能力，整体完成时间通常会比普通视频任务更长
5. 建议始终显式传 `metadata.resolution`

## 4. 建议怎么传

### 4.1 想最终拿到 720p

建议调用：

```json
{
  "model": "sd2.0fast-sr",
  "prompt": "人物自然抬头并微笑",
  "duration": 5,
  "metadata": {
    "resolution": "480p",
    "ratio": "16:9",
    "watermark": false
  }
}
```

### 4.2 想最终拿到 1080p

建议调用：

```json
{
  "model": "seedance2-sr",
  "prompt": "人物缓慢转头并向镜头挥手",
  "duration": 5,
  "metadata": {
    "resolution": "720p",
    "ratio": "16:9",
    "watermark": false
  }
}
```

## 5. 使用建议

- `seedance2` / `seedance2-sr` 建议优先传 `720p` 或 `1080p`
- `sd2.0fast` / `sd2.0fast-sr` 建议优先传 `480p` 或 `720p`，不要传 `1080p`
- 如果希望最终稳定拿到 `1080p`，建议直接提交 `720p`，并使用超分专用模型
- 如果不希望触发超分和额外费用，请继续使用普通模型 `seedance2` 或 `sd2.0fast`

## 6. 常见问题

### 6.1 为什么任务比以前慢

因为专用超分模型会在原始视频生成成功后继续执行一段超分处理，所以整体耗时通常更长。

### 6.2 我拿到的是原始视频还是超分后视频

- 普通模型：拿到原始生成视频
- 专用超分模型：拿到超分后视频

包括：

- `GET /v1/video/generations/{task_id}` 里的 `result_url`
- `GET /v1/videos/{task_id}/content` 返回的视频流

### 6.3 为什么我传了 1080p，没有继续变更高

当前自动超分只处理：

- `480p -> 720p`
- `720p -> 1080p`

已经是 `1080p` 的任务，不会继续向更高分辨率处理。
