# 接口文档重构说明

接口文档我已经重新整理过了，后面统一按这个版本执行。

先把结论说清楚：

- 正式接口文档现在只看 `docs/api/current.md`
- 更新记录统一看 `docs/api/changelog.md`
- 历史文档统一放到 `docs/legacy/`

这次主要改了下面这些地方。

## 1. 文档入口统一了

以前文档分散，接口文档、测试文档、Word 文档、临时说明混在一起，找起来很乱，也不清楚到底该看哪一份。

现在我已经统一收口：

- 正式接入只看 `docs/api/current.md`
- `docs/legacy/` 里的内容全部按历史归档处理
- 新增了 `docs/README.md`，以后先看这里就行

## 2. 阅读顺序重新排了

以前最大的问题不是没有内容，而是入口不清楚，阅读顺序也不清楚。

现在我在文档最前面加了“阅读说明”，直接把路径写清楚了：

- 对话接入看哪里
- 视频接入看哪里
- 真人资产看哪里
- 虚拟人像资产看哪里
- 文件夹看哪里

以后不用整份文档从头翻，按场景直接看对应章节就行。

## 3. 旧版有、主文档里缺的说明补回来了

这次我把之前缺掉的内容补回来了，尤其是对话接入这块：

- DeepSeek 对话接口说明
- 完整对话请求示例
- 请求字段说明
- 流式处理规则
- 常见错误说明

这部分以前旧版里有，主文档里不完整，现在已经补齐。

## 4. 结构上的问题修掉了

以前文档结构本身就有问题，不只是内容问题。

这次我把这些地方一起改了：

- 分页字段说明放回分页章节
- `external_user_id` 隔离规则单独放一节
- 重复编号修正掉了
- 快速开始、通用规范、模型能力的层级重新排过了

## 5. 示例和说明都按接入来写了

这次不是只列接口路径，我把真正接入时会用到的内容一起补齐了：

- 完整接入示例
- 字段怎么填
- 常见错误怎么理解
- 按场景怎么找章节

这样以后看文档，不用再自己猜该填什么，也不用来回翻。

## 6. 历史内容没有删，只是归档了

旧版文档我没有直接删掉，后面如果要追历史写法、核对旧说明、排查老接入差异，还是可以去 `docs/legacy/` 里看。

但从现在开始，统一以主文档为准，不再多份并行。

## 7. 后面统一按这个规则执行

以后你只需要记住这三件事：

1. 正式文档只看 `docs/api/current.md`
2. 历史文档只去 `docs/legacy/` 查
3. 每次文档更新只看 `docs/api/changelog.md`

后面再更新接口文档，我会继续按这套结构维护，不会再回到以前那种到处分散、反复比对的状态。
