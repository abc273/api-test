# 八两 API 接口文档写法规范

## 写法原则

- 用“我们”“你们”表达，不写泛称式称呼。
- 每个接口都必须给完整 `curl`、完整请求体、完整成功响应、完整失败响应。
- 每个变更都必须写清楚是否影响旧调用。
- 不写空话，不写没有结论的描述。
- 如果不确定点来自上游，必须明确写“不确定点来自上游，不是八两 API 本地逻辑”。

## 固定章节

每个接口按下面顺序写：

1. 接口说明
2. 请求地址
3. 请求头
4. Query 参数
5. Body 参数
6. 完整请求示例
7. 成功响应示例
8. 失败响应示例
9. 字段说明
10. 注意事项
11. 兼容性说明

## 发布要求

- 每次发布必须更新 `docs/api/current.md`。
- 每次发布必须更新 `docs/api/changelog.md`。
- 每次新增或调整公开接口，必须更新 `docs/api/endpoints.yaml`。
- 发布前必须执行 `scripts/docs/check-api-docs.sh`。
